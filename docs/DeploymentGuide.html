Windows: feche todas as janelas do PowerShell e abra de novo. Se ainda não funcionar, vá em Configurações → Sistema → Sobre → Configurações avançadas do sistema → Variáveis de ambiente e confirme que C:\Program Files\dotnet\ está no PATH.

Linux: rode source ~/.bashrc ou abra um novo terminal.

3. Configurar a rede do servidor

Antes de testar o sistema, o servidor precisa estar com IP fixo, firewall aberto e acessível pelas máquinas dos admins.

3.1 Definir IP fixo no servidor

Combine com a TI um IP livre que nunca mude. Por exemplo: 10.0.0.5. Esse IP vai ser:

  • O endereço que os admins vão digitar no navegador (http://10.0.0.5:8080);
  • O endereço que o servidor SMTP precisa autorizar para envio;
  • O IP que aparecerá nos logs.

Windows — pelo painel de controle

  1. Pressione Win+R e digite ncpa.cpl.
  2. Botão direito na conexão Ethernet → Propriedades.
  3. Selecione Protocolo IPv4 (TCP/IPv4) → Propriedades.
  4. Marque "Usar o seguinte endereço IP" e preencha (valores de exemplo):
    CampoValor
    Endereço IP10.0.0.5
    Máscara255.255.255.0
    Gateway10.0.0.1 (ou o gateway do CTA)
    DNS10.0.0.1 (ou os do CTA)
  5. Clique OK em todas as janelas.

Windows — pelo PowerShell (alternativa rápida)

# Liste os adaptadores pra descobrir o nome
Get-NetAdapter

# Configure (troque "Ethernet" pelo nome real e ajuste os IPs)
New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 10.0.0.5 -PrefixLength 24 -DefaultGateway 10.0.0.1
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses 10.0.0.1

Linux — Ubuntu Server (netplan)

Edite o arquivo /etc/netplan/00-installer-config.yaml:

sudo nano /etc/netplan/00-installer-config.yaml

Conteúdo esperado (ajuste os IPs):

network:
  version: 2
  ethernets:
    ens33:                       # nome do adaptador (use "ip a" pra descobrir)
      dhcp4: false
      addresses: [10.0.0.5/24]
      routes:
        - to: default
          via: 10.0.0.1
      nameservers:
        addresses: [10.0.0.1, 8.8.8.8]

Salvar (Ctrl+O) → sair (Ctrl+X) → aplicar:

sudo netplan apply
ip a   # confirme que o IP está atribuído
Cuidado ao fazer isso via SSH Se você está conectado via SSH no servidor Linux e mudar o IP, a sessão SSH cai. Sempre faça essa mudança no console físico ou via KVM remoto.

3.2 Abrir as portas 8080 e 5018 no firewall

Sem isso, o navegador dos admins não conecta no painel.

Windows — PowerShell como Administrador

New-NetFirewallRule -DisplayName "5CTA-Painel" -Direction Inbound -LocalPort 8080 -Protocol TCP -Action Allow
New-NetFirewallRule -DisplayName "5CTA-API"    -Direction Inbound -LocalPort 5018 -Protocol TCP -Action Allow

# Conferir
Get-NetFirewallRule -DisplayName "5CTA-*"

Linux — UFW

sudo ufw allow 8080/tcp
sudo ufw allow 5018/tcp
sudo ufw status
Por que abrir a 5018 se ela é interna? Tecnicamente a 5018 é usada só pelo painel chamando localhost, então em produção pura você pode não abrir ela no firewall. Mas em ambiente de testes (depuração, scripts externos), abrir facilita. Em produção final, deixe só a 8080 aberta.

3.3 Testar acesso de outra máquina

Antes de prosseguir, valide que tudo está acessível.

  1. Em qualquer notebook da rede do CTA, abra o navegador.
  2. Digite http://10.0.0.5:8080 (troque pelo IP que você definiu).
  3. Por enquanto o site não está rodando, então vai dar erro de conexão. Isso é esperado. O importante: se aparecer "Não foi possível acessar este site", ainda OK. Se aparecer "Conexão recusada" ou "Tempo limite esgotado", é firewall.
  4. Faça também um ping 10.0.0.5 no CMD do notebook. Deve responder.

4. Primeiro start do sistema

Agora vamos subir os 3 serviços (Int1, Int2 e Int3) pela primeira vez. Tem um script pronto pra isso.

4.1 Rodar o iniciar.ps1

Na raiz do projeto tem o arquivo C:\5CTA\iniciar.ps1 que sobe tudo. Faça assim:

  1. Abrir PowerShell na pasta do projeto

    Vá no Explorer até C:\5CTA\. Mantenha Shift apertado e clique com botão direito em qualquer área vazia da pasta. Escolha "Abrir janela do PowerShell aqui".

  2. Permitir execução do script

    O Windows bloqueia scripts por padrão. Rode uma vez:

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

    (Isso libera só pra essa sessão do PowerShell, não fica permanente.)

  3. Executar o script

    .\iniciar.ps1

    Você vai ver mensagens tipo:

    [1/4] Matando processos antigos nas portas 5018 e 8080...
[2/4] Fazendo backup do banco...
[3/4] Subindo Int1 (Banco/API)... aguardando porta 5018... OK
[4/4] Subindo Int3 (Painel)... aguardando porta 8080... OK
Subindo Int2 (Worker) em background...
Abrindo navegador em http://localhost:8080

    Quando aparecer o navegador com a tela de login, deu certo.

4.2 O admin padrão (admin/Admin@123)

No primeiro start, o sistema cria automaticamente:

  • O arquivo C:\5CTA\Banco\WebAbil8-Sistema_Verificação_dupla.slnx\banco.db (o banco SQLite vazio);
  • 900.000 senhas disponíveis (de 100000 a 999999, exceto triviais);
  • 900.000 códigos de usuário disponíveis;
  • Um administrador padrão: login: admin / senha: Admin@123;
  • Configurações padrão (90 dias retenção, 60s tempo de espera etc).
SEGURANÇA — Troque a senha do admin padrão A senha Admin@123 está documentada (você está lendo agora). Antes de colocar o sistema em uso real, siga o passo §8 desta documentação pra trocar a senha do admin padrão por algo único e forte.

4.3 Ver se está funcionando

  1. No navegador, acesse http://10.0.0.5:8080 (de outra máquina) ou http://localhost:8080 (no próprio servidor).
  2. Tela de login deve aparecer.
  3. Login: admin / senha: Admin@123.
  4. Você cai no Dashboard com 0 acessos. Tudo zerado — é esperado, o sistema acabou de ser criado.

5. Cadastrar e conectar T50M Anviz

Cada T50M tem 2 etapas: primeiro você configura o IP fixo dele no próprio aparelho (menu local), depois cadastra esse IP no painel.

5.1 Configurar IP fixo no T50M

O T50M sai de fábrica com IP 192.168.0.218 (padrão Anviz). Se a sua rede usa essa faixa, talvez já dê pra usar direto. Mas o padrão recomendado é configurar manualmente.

  1. Acesse o menu local do T50M

    No teclado do T50M (já fisicamente instalado e ligado), pressione M/OK.

    Será pedida a senha de admin. Padrão de fábrica: 12345. Se já foi alterada por outra equipe, peça pra TI.

  2. Navegue até System → Comm → Ethernet

    Use as setas ▲▼ pra navegar no menu, M/OK pra confirmar, ESC pra voltar.

    Caminho completo: Menu → System Setting → Comm Setting → Ethernet.

  3. Defina os endereços

    CampoValor exemploO que é
    IP Address10.0.0.10O IP que o T50M vai ter (precisa ser livre na sua LAN)
    Subnet Mask255.255.255.0Igual à do servidor
    Gateway10.0.0.1Mesmo gateway da sua rede
    DNS0.0.0.0Pode deixar zerado (T50M não usa DNS)
    Server IP10.0.0.5IP do servidor onde o sistema roda
    Server Port5010Porta padrão Anviz (não mudar)

    Pressione M/OK em cada campo, digite o valor com o teclado numérico, M/OK pra confirmar.

  4. Salve e reinicie a interface de rede do T50M

    Saia do menu com ESC várias vezes até voltar à tela inicial. O T50M reaplica a config de rede sozinho em alguns segundos.

5.2 Conferir conectividade do servidor com o T50M

Antes de cadastrar no painel, verifique que o servidor enxerga o T50M.

Teste 1 — Ping

# Do servidor
ping 10.0.0.10
# Resposta esperada: 4 pacotes recebidos, 0% de perda.

Teste 2 — Porta TCP 5010

# Windows — testa se a porta está aceitando conexão
Test-NetConnection 10.0.0.10 -Port 5010
# Deve mostrar "TcpTestSucceeded : True"
# Linux
nc -zv 10.0.0.10 5010
# Deve mostrar "Connection to 10.0.0.10 5010 port [tcp/*] succeeded!"

5.4 Erros de conexão T50M

❌ T50M cadastrado, mas fica para sempre "Offline"
Causa provável: Worker não consegue conectar TCP, IP errado, ou Server IP/Port no T50M aponta pra lugar errado.
Checagem 1 — Pinga?
ping 10.0.0.10

Se não responder, problema é de rede. Verifique cabo, IP, máscara, e se T50M e servidor estão na mesma VLAN.

Checagem 2 — Porta 5010 abre?
Test-NetConnection 10.0.0.10 -Port 5010

Se "TcpTestSucceeded: False", o T50M está respondendo ping mas a porta 5010 não está aceitando. Volte no menu local do T50M e confirme o Server Port em System → Comm → Ethernet.

Checagem 3 — Reiniciar o Worker

Pare o Worker e suba de novo. Os logs ajudam:

# Mate todos os processos do Worker
Get-Process | Where-Object { $_.ProcessName -like "*BiometricAcess*" } | Stop-Process -Force

# Volte na pasta e rode o iniciar.ps1
cd C:\5CTA
.\iniciar.ps1

Verifique o log em C:\5CTA\Hardware & Serviço de Background\BiometricAcess.Worker\BiometricAcess.Worker\Logs\. Procure por linhas com Falha ao conectar.

❌ T50M aparece Online mas não responde aos acessos físicos
Causa provável: Modo de verificação local do T50M (FP/PWD) não está habilitado.
Como resolver

No T50M: M/OK → System → Verify → escolha FP + PWD (digital ou senha).

❌ Pessoa cadastrada no painel mas T50M não aceita o ID
Causa provável: A pessoa não foi vinculada a um ambiente, ou a sincronização ainda não rodou.
Como resolver
  • Vá em Pessoas → detalhe da pessoa. Confirme que ela tem status ativo e está em pelo menos um ambiente.
  • Vá em Dispositivos T50 → detalhe do T50. Veja se o contador DigitaisCadastradas aumentou.
  • O SincronizadorT50 roda a cada 10s. Espere 30s e tente acessar de novo.
Múltiplos T50Ms num mesmo ambiente Um ambiente pode ter vários T50Ms (ex: porta de entrada + saída). Quando você vincula uma pessoa ao ambiente, ela é cadastrada em todos os T50Ms do ambiente automaticamente. Capacidade é por T50M (1000 cada).

6. Cadastrar câmeras IP

As câmeras precisam ter ONVIF habilitado e ser cadastradas no sistema. O sistema usa ONVIF pra buscar gravações e RTSP pra preview ao vivo.

6.1 Habilitar ONVIF na câmera

Cada marca tem um caminho um pouco diferente. Em geral:

  1. Acesse o painel web da câmera

    Digite o IP da câmera no navegador (ex: http://192.168.0.50). Login geralmente é admin + a senha definida na instalação.

  2. Procure o menu de ONVIF

    Os caminhos comuns:

    MarcaCaminho típico
    IntelbrasConfiguração → Rede → Avançado → ONVIF
    HikvisionConfiguration → Network → Advanced Settings → Integration Protocol
    DahuaSetup → Network → ONVIF

  3. Crie um usuário dedicado ao ONVIF

    Não use o admin da câmera — crie um usuário separado com permissão de leitura/playback. Anote login e senha.

    Sugestão de credenciais: usuário onvif_5cta, senha forte gerada (use um gerador de senhas).

6.2 Anotar URL RTSP e endpoint ONVIF

Pra cada câmera, anote 3 informações:

O queOnde acharExemplo
URL RTSP No painel web da câmera, seção de Streaming. Cada marca tem um formato. rtsp://onvif_5cta:SenhaForte@192.168.0.50:554/Streaming/Channels/101
Endpoint ONVIF Geralmente http://<IP-CAMERA>:80/onvif/device_service. http://192.168.0.50:80/onvif/device_service
Usuário/senha ONVIF O que você criou em §6.1. onvif_5cta / SenhaForte

Testar o RTSP no VLC antes

Baixe o VLC se ainda não tem. Abra: Mídia → Abrir Stream de Rede → cole a URL RTSP completa. Se aparecer o vídeo ao vivo, está OK pra cadastrar.

6.3 Cadastrar a câmera no painel

Caminho: Menu lateral › Câmeras › botão + Nova Câmera
CampoComo preencher
NomeIdentificação visual. Ex: "Câmera Frontal Sala Op"
AmbienteO ambiente onde a câmera está (precisa estar cadastrado antes — veja UserManual §9)
TipoInterna ou Externa

URL HLS (opcional)Só preencha se configurou MediaMTX (ver §6.4)
Endereço ONVIFO endpoint completo

Clique em Cadastrar. Pronto — a próxima entrada liberada nesse ambiente já vai gerar uma URL de gravação que aparece no Histórico.

6.4 (Opcional) Streaming HLS pra ver ao vivo

RTSP não roda direto no navegador moderno. Pra ver a câmera ao vivo dentro do painel, é preciso um servidor de mídia que converte RTSP → HLS. O mais simples é o MediaMTX.

Instalar o MediaMTX (Linux)

cd /opt
sudo wget https://github.com/bluenviron/mediamtx/releases/download/v1.8.5/mediamtx_v1.8.5_linux_amd64.tar.gz
sudo tar xzf mediamtx_v1.8.5_linux_amd64.tar.gz
sudo rm mediamtx_v1.8.5_linux_amd64.tar.gz

Configurar uma câmera

Edite /opt/mediamtx.yml e adicione (uma entrada por câmera):

paths:
  cam-frontal:
    source: rtsp://onvif_5cta:SenhaForte@192.168.0.50:554/Streaming/Channels/101
    sourceProtocol: tcp
  cam-corredor:
    source: rtsp://onvif_5cta:SenhaForte@192.168.0.51:554/Streaming/Channels/101
    sourceProtocol: tcp

Iniciar

cd /opt
./mediamtx
# Por padrão escuta na porta 8888 (HLS)

A URL HLS pra colar no painel fica:

http://<IP-DO-MEDIAMTX>:8888/cam-frontal/index.m3u8

Pra rodar como serviço (não cair se reiniciar), crie /etc/systemd/system/mediamtx.service:

[Unit]
Description=MediaMTX
After=network.target

[Service]
ExecStart=/opt/mediamtx /opt/mediamtx.yml
Restart=always
User=root

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable --now mediamtx

7. Configurar envio de email pelo SMTP do CTA

Quando você cadastra uma pessoa, o sistema tenta enviar ID + senha por email automaticamente. Pra isso precisa do servidor SMTP do CTA configurado.

7.1 Coletar dados do servidor SMTP

Erros comuns no SMTP

❌ "Authentication failed" / "535 5.7.8"
Causa: usuário/senha errados, ou a conta exige App Password.
Como resolver
  1. Teste o usuário e senha em um cliente de email normal (Thunderbird, Outlook) configurando IMAP/SMTP. Se não funciona lá, problema é com a conta — fale com a TI.
  2. Se for conta moderna (Microsoft 365, Outlook.com pessoal, Gmail), pode exigir App Password (uma senha específica gerada pra apps externos). Peça pra TI gerar.
  3. Confira se sua senha tem caracteres especiais que foram mal escapados. No PowerShell, senhas com $, ", ! dão problema — use sempre aspas simples ou variável intermediária: 
    $senha = 'M1nh@$enhaSecreta'
[System.Environment]::SetEnvironmentVariable("SMTP_PASS", $senha, "Machine")
❌ "Connection timeout" no SMTP
Causa: firewall bloqueando saída na porta 587 ou servidor SMTP fora do ar.
Como resolver
# Do servidor, teste a conexão TCP
Test-NetConnection smtp.ebmail.eb.mil.br -Port 587

Se "TcpTestSucceeded: False", peça pra TI liberar saída pra esse host/porta no firewall corporativo.

❌ "TLS handshake failed"
Causa: certificado SSL do servidor SMTP autoassinado ou desatualizado.
Como resolver

Verifique se a porta é realmente 587 (STARTTLS) e não 465 (SSL direto). Se o CTA usa certificado autoassinado, peça pra TI adicionar o CA root deles no trust store do Windows ou Linux do servidor.

8. Cadastrar administradores no banco

Por decisão de segurança, criar/editar admin e trocar senha não é feito pela tela. É feito direto no banco SQLite com um cliente como o DBeaver.

8.1 Por que é via banco e não pela tela

Decisão de arquitetura Se a UI permitisse criar admin, qualquer admin comprometido criaria outro admin para si — escalada de privilégio sem rastreabilidade. Forçar via banco implica que quem cadastra admin já tem acesso físico ao servidor. Quem tem acesso ao servidor é uma pessoa de confiança da TI.

Você precisa instalar um cliente SQLite. Os 2 mais usados:

  • DBeaver Community (recomendado) — dbeaver.io/download
  • DB Browser for SQLite (mais simples) — sqlitebrowser.org/dl

8.2 Gerar o hash da senha

A senha nunca pode ir em texto puro pro banco — tem que ser hash BCrypt. O sistema vem com um pequeno programa que gera esse hash. Ele fica em:

C:\5CTA\docs\scripts\GerarHash\

  1. Abrir PowerShell na pasta

    No Explorer, vá em C:\5CTA\docs\scripts\GerarHash. Shift + botão direito → "Abrir janela do PowerShell aqui".

  2. Rodar o gerador

    Forma 1 — passando a senha como argumento (rápido):

    dotnet run -- "MinhaSenha@Forte123"

    Forma 2 — interativo (não fica no histórico):

    dotnet run
# vai pedir a senha

  3. Copiar o hash

    Saída esperada:

    Hash BCrypt (fator 10):
$2a$10$abcd1234efgh5678ijkl9012mnop3456qrst7890uvwxyzABCDEF

    Selecione e copie o hash todo (começa com $2a$10$). Esse é o valor que vai no campo senhaHash do INSERT.

8.3 Abrir o banco no DBeaver

  1. Instale e abra o DBeaver

    Baixe e instale do site oficial. Abra o programa.

  2. Nova conexão SQLite

    • Menu Database → New Database Connection
    • Escolha SQLite e clique Next
    • No campo Path, navegue até: C:\5CTA\Banco\WebAbil8-Sistema_Verificação_dupla.slnx\banco.db
    • Clique Test Connection — deve dizer "Connected"
    • Clique Finish

  3. Abrir editor SQL

    Botão direito na conexão criada → SQL Editor → New SQL script. Aparece uma aba em branco com cursor pronto pra digitar SQL.

8.4 Executar o INSERT (com commit!)

Cole o INSERT abaixo no editor, ajustando os valores entre aspas:

INSERT INTO administrador (login, senhaHash, nomeCompleto, cpf, email, cargo, telefone, dataCriacao)
VALUES (
    'capitao.silva',                              -- login curto, único no sistema
    '$2a$10$<hash gerado em §8.2>',                -- COLE O HASH AQUI
    'Cap José da Silva',                          -- nome completo pra exibição
    '12345678901',                                -- CPF apenas dígitos
    'jose.silva@ebmail.eb.mil.br',                -- email
    'Capitão',                                    -- cargo/patente
    '11987654321',                                -- telefone apenas dígitos
    datetime('now')                               -- data atual UTC
);

  1. Selecionar a linha do INSERT

    Use o mouse pra selecionar o INSERT inteiro (do INSERT até o );).

  2. Executar

    Pressione Ctrl+Enter. Na parte inferior aparece "1 row affected" — o INSERT entrou na transação local.

  3. ⚠ COMMITAR

    Não pula esse passo! Sem commit, o INSERT fica só na transação local do DBeaver e o painel não vê o novo admin. Sintoma típico: "Inseri o admin mas o login não funciona."

    Pressione Ctrl+Shift+C. Ou clique no botão de commit na barra de transações (ícone verde 💾).

    Alternativa: ative auto-commit. Botão direito na conexão → Edit Connection → Transactions → Auto-commit. Aí qualquer execução commita automaticamente.

  4. Testar

    Logue no painel com o novo login/senha. Não precisa reiniciar nada — o admin já está disponível imediatamente.

8.5 Trocar senha de admin existente

Mesma lógica do INSERT, mas com UPDATE.

  1. Gere um novo hash (§8.2) pra senha nova.
  2. No DBeaver SQL editor, execute: 
    UPDATE administrador
SET senhaHash = '$2a$10$<novo hash>'
WHERE login = 'capitao.silva';
  3. Commit (Ctrl+Shift+C).
  4. O admin precisa logar de novo — sessões antigas continuam válidas até expirar (8h). Pra forçar logout: reinicie o Int1.

Inativar admin (sem apagar)

A tabela administrador não tem coluna de status. Pra impedir login, troque a senha pra algo aleatório que ninguém saiba:

# Gera um GUID e usa como senha (descartável)
cd C:\5CTA\docs\scripts\GerarHash
dotnet run -- "$(New-Guid)"

Pegue o hash e UPDATE no banco. Como ninguém sabe a senha (foi descartada na hora), o admin não loga mais.

Apagar definitivamente

DELETE FROM administrador WHERE login = 'admin_antigo';

Logs antigos ficam com adminId órfão. No painel aparece como Admin {id}. É proposital — auditoria não some.

Trocar a senha do admin padrão (admin/Admin@123)

Faça isso antes de colocar o sistema em uso real:

cd C:\5CTA\docs\scripts\GerarHash
dotnet run -- "<Senha forte nova aqui>"
UPDATE administrador
SET senhaHash = '$2a$10$<hash colado aqui>'
WHERE login = 'admin';

Commit (Ctrl+Shift+C).

10. Backup e manutenção

10.1 Backup automático do banco

O Int1 já faz backup do banco ao iniciar (cópia em banco_backup.db na mesma pasta). Pra backup adicional (sugerido: diário em outro disco/local):

Windows — Task Scheduler

Crie o script C:\5CTA\backup.ps1:

$origem  = "C:\5CTA\Banco\WebAbil8-Sistema_Verificação_dupla.slnx\banco.db"
$destino = "D:\Backups\5CTA\banco-$(Get-Date -f 'yyyyMMdd-HHmm').db"
$pasta   = Split-Path $destino -Parent
if (-not (Test-Path $pasta)) { New-Item -ItemType Directory -Path $pasta -Force | Out-Null }
Copy-Item $origem $destino
# Mantém só os últimos 30 backups
Get-ChildItem $pasta -Filter "banco-*.db" |
    Sort-Object LastWriteTime -Descending |
    Select-Object -Skip 30 |
    Remove-Item

Agendar pra rodar todo dia às 02:00:

schtasks /create /tn "5CTA-Backup" /sc daily /st 02:00 /tr "powershell.exe -ExecutionPolicy Bypass -File C:\5CTA\backup.ps1"

Linux — cron

sudo crontab -e
# Adicionar:
0 2 * * * cp /opt/5cta/Banco/WebAbil8-Sistema_Verificação_dupla.slnx/banco.db /var/backups/5cta/banco-$(date +\%Y\%m\%d).db

11. Bônus — Implantação via Docker

Em vez de instalar .NET no servidor, você pode rodar o sistema todo em containers Docker. Útil pra padronizar implantação em várias unidades militares.

Quando vale a pena

  • Você vai testar antes de mexer no servidor de produção
  • Tem várias máquinas com versões diferentes do .NET
  • Quer rollback rápido entre versões
  • Vai implantar em mais de uma unidade do CTA

Quando NÃO vale

  • Comunicação Worker ↔ T50M via TCP fica mais complicada (precisa --network host no Linux, ou IP de rota explícita no Windows)
  • Equipe da TI sem familiaridade com Docker — manutenção fica mais difícil

12. Quando algo der errado — Troubleshooting global

Resumo dos problemas mais comuns separados por sintoma. Procure o seu pelo sintoma.

Sintoma: o painel não abre de fora do servidor

Sintoma: erro de conexão / "site não disponível"
  • Firewall: revise §3.2.
  • IP errado: use ipconfig (Win) ou ip a (Linux) no servidor pra confirmar.
  • Serviço não está rodando: Get-Service "5CTA-Painel" deve dizer Running.
  • Serviço escutando só em localhost: verifique C:\5CTA\PainelWeb\Frontend\Properties\launchSettings.json — applicationUrl tem que ser http://0.0.0.0:8080, não http://localhost:8080.

Sintoma: T50M não fica Online

Veja §5.4. Em ordem: ping → testar porta 5010 → reiniciar Worker → ver log do Worker.

Sintoma: Email não chega

Veja §7.3. Confira no painel a banner de status do envio na modal de cadastro. Se falhou, olhe os logs:

Get-Content "C:\5CTA\Banco\WebAbil8-Sistema_Verificação_dupla.slnx\Logs\*.log" -Tail 50
# Procure por "SMTP falhou" ou "[FALLBACK]"

Sintoma: gravação de câmera não aparece no histórico

  • Câmera está cadastrada no ambiente da tentativa?
  • Endpoint ONVIF está correto? Teste com curl http://<IP-CAMERA>:80/onvif/device_service — deve retornar XML.
  • Aumente o Tempo de espera em Configurações de 60s pra 90/120s (câmeras antigas levam mais tempo pra liberar a gravação).
  • A câmera grava por movimento ou contínuo? Sem gravação ativa, não há nada pra puxar.

Sintoma: novo admin inserido no banco mas o login não funciona

99% das vezes é falta de commit no DBeaver. Volte em §8.4, passo 3, e pressione Ctrl+Shift+C.

Sintoma: contador "DigitaisCadastradas" do T50M não cresce
